# ClawFin Briefing Integration Skill

Use this guide when an external automation tool or LLM agent needs to produce daily or weekly ClawFin transaction status messages.

## What ClawFin Provides

ClawFin is the source of truth for already-synced financial data. External agents should call ClawFin for summarized context instead of asking for SimpleFIN credentials or connecting to banks directly.

The briefing API can return:
- Structured transaction context for an external LLM.
- A ready-to-send plain-English summary generated by ClawFin's configured AI provider.

## Authentication

Set a dedicated automation token in the ClawFin backend environment:

```bash
CLAWFIN_AUTOMATION_TOKEN=<random-token>
```

Send it on every briefing request:

```http
X-ClawFin-Automation-Token: <random-token>
```

Do not use SimpleFIN access URLs, bank passwords, frontend JWTs, or AI provider API keys in external prompts.

## Recommended Automation Flow

1. Trigger from cron, n8n, Windmill, or another scheduler.
2. Optionally call ClawFin's SimpleFIN sync endpoint first:
   ```http
   POST /api/import/simplefin/sync
   ```
   The sync response includes `stale_accounts` when ClawFin sees accounts with stale data, missing SimpleFIN presence, or a sync error.
3. Call the briefing endpoint:
   ```http
   POST /api/briefings/transactions
   ```
4. Send either `summary` or externally generated prose to Telegram, email, Signal, Matrix, or another notification target.

For the first version, scheduling and notification delivery live outside ClawFin.

## Endpoint

```http
POST /api/briefings/transactions
Content-Type: application/json
X-ClawFin-Automation-Token: <token>
```

Request body:

```json
{
  "period": "daily",
  "mode": "generate",
  "end_date": "2026-04-25",
  "include_transactions": false,
  "max_transactions": 25,
  "redact_merchants": false
}
```

Fields:
- `period`: `daily` or `weekly`.
- `mode`: `context` returns JSON only; `generate` returns JSON plus `summary`.
- `end_date`: optional ISO date. If omitted, ClawFin uses today's server date.
- `include_transactions`: include recent transaction rows in `context.transactions`.
- `max_transactions`: cap included transactions from 0 to 100.
- `redact_merchants`: replace merchant names with `REDACTED` before returning context or sending it to the LLM.

## Daily Context Example

```bash
curl -s http://localhost:8000/api/briefings/transactions \
  -H "Content-Type: application/json" \
  -H "X-ClawFin-Automation-Token: $CLAWFIN_AUTOMATION_TOKEN" \
  -d '{
    "period": "daily",
    "mode": "context",
    "include_transactions": true,
    "redact_merchants": true
  }'
```

Use `mode: "context"` when a third-party LLM will write the final text. Prefer `redact_merchants: true` unless merchant names are needed.

## Weekly Generated Summary Example

```bash
curl -s http://localhost:8000/api/briefings/transactions \
  -H "Content-Type: application/json" \
  -H "X-ClawFin-Automation-Token: $CLAWFIN_AUTOMATION_TOKEN" \
  -d '{
    "period": "weekly",
    "mode": "generate",
    "include_transactions": false
  }'
```

Response shape:

```json
{
  "period": "weekly",
  "mode": "generate",
  "range": {"start": "2026-04-19", "end": "2026-04-25", "days": 7},
  "context": {
    "totals": {
      "income": 2500.0,
      "expenses": 420.75,
      "net_cash_flow": 2079.25,
      "previous_period_expenses": 390.12,
      "expenses_delta_pct": 7.9
    },
    "counts": {
      "transactions": 18,
      "pending": 2,
      "uncategorized": 1
    }
  },
  "summary": "Weekly ClawFin status..."
}
```

## External LLM Prompt

When using `mode: "context"`, pass the returned `context` with this instruction:

```text
Write a concise ClawFin daily or weekly transaction status message.
Use only the supplied JSON. Do not invent numbers.
Include total spend, income, net cash flow, pending or uncategorized counts if non-zero, the largest spending category, unusual spend, and any reconnect nudge.
Do not provide investment, tax, legal, or debt advice.
Keep it under 10 short lines and make it suitable for a private notification.
```

## Privacy Rules

- Never ask the user for SimpleFIN credentials, bank credentials, or the SimpleFIN access URL.
- Do not send full raw transaction history unless `include_transactions` is explicitly needed.
- Prefer `mode: "context"` plus `redact_merchants: true` for third-party LLMs.
- Prefer `mode: "generate"` when ClawFin is configured with a local model such as Ollama.
- Treat summaries as sensitive financial data. Use private notification channels.

## Failure Handling

- `401 Missing automation token`: add the `X-ClawFin-Automation-Token` header.
- `401 Invalid automation token`: check the token value against `CLAWFIN_AUTOMATION_TOKEN`.
- `503 Automation token not configured`: set `CLAWFIN_AUTOMATION_TOKEN` and restart the backend.
- `400 AI provider not configured`: use `mode: "context"` or configure ClawFin AI settings.
- `502 AI provider request failed`: retry later or fall back to `mode: "context"`.
- Empty ledger: send a short “no transactions found for this period” notification.
- `account_freshness.reconnect_nudge: true`: include a reconnect/sync check in the notification.
- `stale_reason: "missing_from_simplefin_response"`: the account previously existed locally but was absent from the latest SimpleFIN response.
- `stale_reason: "simplefin_auth_error"`: the SimpleFIN access URL or upstream authorization likely needs attention.
- `stale_reason: "simplefin_sync_error"`: the latest SimpleFIN request failed; retry and inspect backend logs.
- `stale_reason: "simplefin_sync_stale"`: the account is present, but ClawFin has not recorded a successful SimpleFIN sync within `CLAWFIN_SIMPLEFIN_STALE_DAYS`.
- `stale_reason: "never_synced"`: the account has no recorded SimpleFIN sync timestamp yet.
